# Diabetes 30-Day Hospital Readmission Prediction

> Predicting whether a diabetic patient will be readmitted to the hospital within 30 days of discharge, using 10 years of clinical billing data from 130 US hospitals (1999–2008).

**Course:** CSP 571 — Data Preparation & Analysis | IIT | 2026  
**Team:** Nithiya · Deepesh · Pavan  
**Dataset:** [UCI Diabetes 130-US Hospitals](https://archive.ics.uci.edu/dataset/296/diabetes+130-us+hospitals+for+years+1999-2008) — 101,766 encounters · 50 features · CC BY 4.0

---

## Overview

Unplanned 30-day hospital readmissions impose a significant clinical and economic burden — diabetic inpatients alone cost the US healthcare system over **$250 million** in avoidable readmissions in 2011. This project builds, tunes, and ensembles four machine-learning classifiers on real hospital billing records to flag high-risk patients before discharge.

The end-to-end pipeline is implemented in **R** (primary) with **Python** used exclusively for SHAP explainability. It covers data auditing, preprocessing, exploratory analysis, feature engineering, hyperparameter tuning, stacking ensemble, threshold optimization, and statistical significance testing.

## Results

> Evaluated on a held-out 30% test set (stratified split, seed = 42).  
> XGBoost / LightGBM / Ensemble use F1-optimal threshold; LR / RF use default 0.5.

| Model | AUC | Precision | Recall | F1 | Notes |
|---|---|---|---|---|---|
| Logistic Regression | 0.615 | 0.121 | 0.530 | 0.197 | Baseline |
| Random Forest | 0.624 | 0.188 | 0.181 | 0.185 | Severe overfit (CV AUC 0.973) |
| XGBoost | 0.647 | 0.139 | 0.569 | 0.223 | Threshold optimized |
| LightGBM | **0.649** | — | — | — | Best single-model AUC |
| **Stacking Ensemble** | — | ~0.16 | **0.443** | — | Best clinical recall trade-off |

**Best clinical deployment choice:** Stacking Ensemble at threshold 0.14 — maximizes recall (catches most true readmissions) while keeping AUC competitive. DeLong test confirmed RF is significantly worse than the ensemble (p < 0.0001).

## Key Findings

- **Discharge disposition** is the strongest predictor — where a patient goes after discharge matters more than nearly any in-hospital feature (top SHAP, mean |SHAP| = 0.246)
- **Prior inpatient visits** (≥3) identify a high-risk subgroup with 2–3× the average readmission rate (mean |SHAP| = 0.133)
- **Data leakage fix** — removing deceased/hospice patients (discharge IDs 11, 13, 14, 19, 20, 21) shifts the positive rate from 8.8% → 7.1%, correctly reflecting the actual at-risk population
- **SMOTE** applied to the training set only (K=5) to address the 13:1 class imbalance without contaminating test evaluation
- **Performance ceiling ~0.65 AUC** is a data constraint, not a modeling failure — billing records lack vitals, HbA1c time series, and lab results that would be needed to push further

## Pipeline Overview

```
diabetic_data.csv  (101,766 rows × 50 cols)
        │
        ▼  01_data_audit.R
           → missing_audit.csv, cardinality.csv
        │
        ▼  02_preprocessing.R
           → diabetes_clean.csv  (71,518 rows after dedup)
           • Remove deceased/hospice (discharge IDs 11,13,14,19,20,21)
           • Deduplicate: first encounter per patient only
           • ICD-9 codes → 9 diagnosis groups
           • Age brackets → numeric midpoints
           • Binary target: readmitted_binary = (readmitted == "<30")
           • Positive rate: 7.1%  (13:1 class imbalance)
        │
        ▼  03_eda.R
           → figures 01–16, statistical_tests.csv
        │
        ▼  04_feature_engineering.R
           → diabetes_featured.csv  (+8 derived features, 55 total)
           • high_utilizer, polypharmacy, total_visits, diab_primary
           • any_change, age_×_inpatient, med_×_diagnoses, emergency_ratio
           • Charlson comorbidity proxy, glucose control flags
        │
        ▼  05_modeling.R  (70/30 stratified split + SMOTE K=5 on train only)
           • Logistic Regression  — glmnet, lambda grid
           • Random Forest        — mtry grid, ntree = 200
           • XGBoost              — 108-combo grid (max_depth × eta), 5-fold CV + early stopping
           • LightGBM             — 27-combo grid (num_leaves × lr), 5-fold CV + early stopping
        │
        ▼  06_ensemble.R
           → Stacking Ridge meta-learner on 5-fold OOF probabilities
           → Threshold optimization (F1-optimal & recall ≥ 70%)
           → optimal_thresholds.csv
        │
        ▼  07_evaluation.R
           → model_comparison.csv, figures 17–21
           → AUC · PR-AUC · Precision · Recall · Specificity · NPV · F1
           → DeLong significance test: Ensemble vs each base model
        │
        ▼  python/shap_analysis.py
           → figures 22–24, shap_values.csv
           → SHAP TreeExplainer on XGBoost JSON model
```

## Limitations & Future Work

- All models are constrained to ~0.65 AUC because billing records lack clinical data (vitals, HbA1c time series, lab values) — integrating EHR data would likely break this ceiling
- Low precision (~16%) at the recall-optimized threshold means roughly 6 false positives per true positive — acceptable for a screening tool, but care coordinators need to be calibrated accordingly
- Data from 1999–2008 may not reflect current clinical pathways, drug protocols, or ICD coding standards
- Platt scaling calibration should be applied before clinical deployment — raw model probabilities (especially RF) are poorly calibrated
- Future: deploy ensemble at threshold 0.14, monitor precision/recall drift with Evidently AI, and explore fairness audits across age and race subgroups
